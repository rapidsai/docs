Returns a vector of strings columns for each matching group specified in the given regular expression pattern.

All the strings for the first group will go in the first output column; the second group go in the second column and so on. Null entries are added if the string does match.

Any null string entries return corresponding null output column entries.

Example:
s = ["a1","b2","c3"]
r = extract(s,"([ab])(\\d)")
r is now [["a","b",null],
["1","2",null]]

See the Regex Features page for details on patterns supported by this API.

Parameters
stringsStrings instance for this operation.
patternThe regular expression pattern with group indicators.
mrDevice memory resource used to allocate the returned table's device memory.
Returns
Columns of strings extracted from the input column.

Returns a new strings column that contains substrings of the strings in the provided column using unique ranges for each string.

The character positions to retrieve in each string are specified in the starts and stops integer columns. If a start position is outside a string's length, an empty string is returned for that entry. If a stop position is past the end of a string's length, the end of the string is used for stop position for that string. Any stop position value set to -1 will indicate to use the end of the string as the stop position for that string.

Null string entries will return null output string entries.

The starts and stops column must both be the same integer type and must be the same size as the strings column.

Example:
s = ["hello", "goodbye"]
starts = [ 1, 2 ]
stops = [ 5, 4 ]
r = substring_from(s,starts,stops)
r is now ["ello","od"]
Exceptions
cudf::logic_errorif starts or stops is a different size than the strings column.
cudf::logic_errorif starts and stops are not same integer type.
cudf::logic_errorif starts or stops contains nulls.
Parameters
stringsStrings column for this operation.
startsFirst character positions to begin the substring.
stopsLast character (exclusive) positions to end the substring.
mrDevice memory resource used to allocate the returned column's device memory.
Returns
New strings column with sorted elements of this instance.

Slices a column of strings by using a delimiter as a slice point.

Returns a column of strings after searching for delimiter count number of times in the source strings from left to right if count is positive or from right to left if count is negative. If count is positive, it returns a substring from the start of the source strings up until count occurrence of the @delimiter not including the delimiter. If count is negative, it returns a substring from the start of the count occurrence of the @delimiter in the source strings past the delimiter until the end of the string.

The search for @delimiter in strings is case sensitive. If the row value of strings is null, the row value in the output column will be null. If the count is 0 or if delimiter is invalid or empty, every row in the output column will be an empty string. If the column value for a row is empty, the row value in the output column will be empty. If count occurrences of delimiter isn't found, the row value in the output column will be the row value from the input strings column.

Example:
in_s = ['www.nvidia.com', null, 'www.google.com', '', 'foo']
r = slice_strings(in_s, '.', 1)
r = ['www', null, 'www', '', 'foo']
in_s = ['www.nvidia.com', null, 'www.google.com', '', 'foo']
r = slice_strings(in_s, '.', -2)
r = ['nvidia.com', null, 'google.com', '', 'foo']
Parameters
stringsStrings instance for this operation.
delimiterUTF-8 encoded string to search for in each string.
countNumber of times to search for delimiter in each string. If the value is positive, delimiter is searched from left to right; else, it is searched from right to left.
mrResource for allocating device memory.
Returns
New strings column containing the substrings.

Slices a column of strings by using a delimiter column as slice points.

Returns a column of strings after searching the delimiter defined per row from delimiter_strings count number of times in the source strings from left to right if count is positive or from right to left if count is negative. If count is positive, it returns a substring from the start of the source strings up until count occurrence of the delimiter for that row not including that delimiter. If count is negative, it returns a substring from the start of the count occurrence of the delimiter for that row in the source strings past the delimiter until the end of the string.

The search for delimiter_strings in strings is case sensitive. If the count is 0, every row in the output column will be an empty string. If the row value of strings is null, the row value in the output column will be null. If the row value from delimiter_strings is invalid or null, the row value in the output column will be an empty string. If the row value from delimiter_strings or the column value for a row is empty, the row value in the output column will be empty. If count occurrences of delimiter isn't found, the row value in the output column will be the row value from the input strings column.

Example:
in_s = ['www.nvidia.com', null, 'www.google.com', 'bar', 'foo..bar....goo']
delimiters = ['.', '..', '', null, '..']
r = slice_strings(in_s, delimiters, 2)
r = ['www.nvidia', null, '', '', 'foo..bar']
in_s = ['www.nvidia.com', null, 'www.google.com', '', 'foo..bar....goo', 'apache.org']
delimiters = ['.', '..', '', null,'..', '.']
r = slice_strings(in_s, delimiters, -2)
r = ['nvidia.com', null, '', '', '..goo', 'apache.org']
Exceptions
cudf::logic_errorif the number of rows in strings and @delimiter_strings do not match.
Parameters
stringsStrings instance for this operation.
delimiter_stringsUTF-8 encoded string for each row.
countNumber of times to search for delimiter in each string. If the value is positive, delimiter is searched from left to right; else, it is searched from right to left.
