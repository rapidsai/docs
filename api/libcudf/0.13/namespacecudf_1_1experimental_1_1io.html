◆ compression_type

enum cudf::experimental::io::compression_type
strong

Compression algorithms.

Enumerator
NONE

No compression.

AUTO

Automatically detect or select compression format.

SNAPPY

Snappy format, using byte-oriented LZ77.

GZIP

GZIP format, using DEFLATE algorithm.

BZIP2

BZIP2 format, using Burrows-Wheeler transform.

BROTLI

BROTLI format, using LZ77 + Huffman + 2nd order context modeling.

ZIP

ZIP format, using DEFLATE algorithm.

XZ

XZ format, using LZMA(2) algorithm.

◆ io_type

enum cudf::experimental::io::io_type
strong

Data source or destination types.

Enumerator
FILEPATH

Input/output is a file path.

HOST_BUFFER

Input/output is a buffer in host memory,.

ARROW_RANDOM_ACCESS_FILE

Input/output is an arrow::io::RandomAccessFile.

VOID

Input/output is nothing. No work is done. Useful for benchmarking.

USER_SINK

Input/output is handled by a custom user class.

◆ find_all_from_set()

template<class T >
cudf::size_type cudf::experimental::io::find_all_from_set ( const char * h_data ,
size_t h_size ,
const std::vector< char > & keys ,
uint64_t result_offset ,
T * positions
)

Searches the input character array for each of characters in a set. Sums up the number of occurrences. If the 'positions' parameter is not void*, positions of all occurrences are stored in the output device array.

Does not load the entire file into the GPU memory at any time, so it can be used to parse large files. Output array needs to be preallocated.

Parameters
[in] h_data Pointer to the input character array
[in] h_size Number of bytes in the input array
[in] keys Vector containing the keys to count in the buffer
[in] result_offset Offset to add to the output positions
[out] positions Array containing the output positions
Returns
cudf::size_type total number of occurrences

Begin the process of writing an ORC file in a chunked/stream form.

Begin the process of writing an ORC file in a chunked/stream form.

The intent of the write_orc_chunked_ path is to allow writing of an arbitrarily large / arbitrary number of rows to an ORC file in multiple passes.

The following code snippet demonstrates how to write a single ORC file containing one logical table by writing a series of individual cudf::tables.

#include <cudf.h>
...
std::string filepath = "dataset.orc" ;
cudf::experimental::io::write_orc_chunked_args args{ cudf::sink_info (filepath), table->view()};
...
auto state = cudf::write_orc_chunked_begin(args);
cudf::write_orc_chunked (table0, state);
cudf::write_orc_chunked (table1, state);
...
cudf_write_orc_chunked_end(state);
Parameters
[in] args Settings for controlling writing behavior
[in] mr Optional resource to use for device memory allocation
