Compression algorithms.

Enumerator
NONE 

No compression.

AUTO 

Automatically detect or select compression format.

SNAPPY 

Snappy format, using byte-oriented LZ77.

GZIP 

GZIP format, using DEFLATE algorithm.

BZIP2 

BZIP2 format, using Burrows-Wheeler transform.

BROTLI 

BROTLI format, using LZ77 + Huffman + 2nd order context modeling.

ZIP 

ZIP format, using DEFLATE algorithm.

XZ 

XZ format, using LZMA(2) algorithm.

Searches the input character array for each of characters in a set. Sums up the number of occurrences. If the 'positions' parameter is not void*, positions of all occurrences are stored in the output device array.

Does not load the entire file into the GPU memory at any time, so it can be used to parse large files. Output array needs to be preallocated.

Parameters

[in]h_sizeNumber of bytes in the input array
[in]keysVector containing the keys to count in the buffer
[in]result_offsetOffset to add to the output positions
[out]positionsArray containing the output positions
Returns
cudf::size_type total number of occurrences

Begin the process of writing a parquet file in a chunked/stream form.

Begin the process of writing a parquet file in a chunked/stream form.

The intent of the write_parquet_chunked_ path is to allow writing of an arbitrarily large / arbitrary number of rows to a parquet file in multiple passes.

The following code snippet demonstrates how to write a single parquet file containing one logical table by writing a series of individual cudf::tables.

#include <cudf.h>
...
std::string filepath = "dataset.parquet";
cudf::experimental::io::write_parquet_chunked_args args{cudf::sink_info(filepath), table->view()};
...
auto state = cudf::write_parquet_chunked_begin(args);
cudf::write_parquet_chunked(table0, state);
cudf::write_parquet_chunked(table1, state);
...
cudf_write_parquet_chunked_end(state);
Parameters
[in]argsSettings for controlling writing behavior
[in]mrOptional resource to use for device memory allocation
Returns
pointer to an anonymous state structure storing information about the chunked write. this pointer must be passed to all subsequent write_parquet_chunked() and write_parquet_chunked_end() calls.
