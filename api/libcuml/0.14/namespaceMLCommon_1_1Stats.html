Enumeration Type Documentation

â—† HistType

enum MLCommon::Stats::HistType

Types of support histogram implementations

Enumerator
HistTypeSmemBits1

shared mem atomics but with bins to be 1b int's

HistTypeSmemBits2

shared mem atomics but with bins to be 2b int's

HistTypeSmemBits4

shared mem atomics but with bins to be 4b int's

HistTypeSmemBits8

shared mem atomics but with bins to ba 1B int's

HistTypeSmemBits16

shared mem atomics but with bins to be 2B int's

HistTypeGmem

use only global atomics

HistTypeSmem

uses shared mem atomics to reduce global traffic

HistTypeSmemMatchAny

uses shared mem atomics with match_any intrinsic to further reduce shared memory traffic. This can only be enabled on Volta and later architectures. If one tries to enable this for older arch's, it will fall back to HistTypeSmem .

Note
This is to be used only when the input dataset leads to a lot of repetitions in a given warp, else, this algo can be much slower than HistTypeSmem !
HistTypeSmemHash

builds a hashmap of active bins in shared mem

HistTypeAuto

decide at runtime the best algo for the given inputs

template<typename T , int TPB = 512>
void MLCommon::Stats::minmax ( const T * data ,
const unsigned * rowids ,

int ncols ,
int row_stride ,
T * globalmin ,
T * globalmax ,
T * sampledcols ,

Computes min/max across every column of the input matrix, as well as optionally allow to subsample based on the given row/col ID mapping vectors.

Template Parameters
T the data type
TPB number of threads per block
Parameters
data input data
rowids actual row ID mappings. It is of length nrows. If you want to skip this index lookup entirely, pass nullptr
colids actual col ID mappings. It is of length ncols. If you want to skip this index lookup entirely, pass nullptr
nrows number of rows of data to be worked upon. The actual rows of the input "data" can be bigger than this!
ncols number of cols of data to be worked upon. The actual cols of the input "data" can be bigger than this!
row_stride stride (in number of elements) between 2 adjacent columns
globalmin final col-wise global minimum (size = ncols)
globalmax final col-wise global maximum (size = ncols)
sampledcols output sampled data. Pass nullptr if you don't need this
stream cuda stream
Note
This method makes the following assumptions:
  1. input and output matrices are assumed to be col-major
  2. ncols is small enough to fit the whole of min/max values across all cols in shared memory
