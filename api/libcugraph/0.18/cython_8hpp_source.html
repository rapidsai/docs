52 // "container" for a graph type instance which insulates the owner from the
53 // specifics of the actual graph type. This is intended to be used in Cython
54 // code that only needs to pass a graph object to another wrapped C++ API. This
55 // greatly simplifies the Cython code since the Cython definition only needs to
56 // define the container and not the various individual graph types in Cython.

78  // of a cython wrapper simply for passing a templated instantiation of a

82  // to an instance are not supported and these methods are deleted.
